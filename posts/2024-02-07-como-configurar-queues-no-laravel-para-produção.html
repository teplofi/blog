Como configurar queues no Laravel para produção

Este é um lembrete rápido sobre Como configurar queues no Laravel para produção. São notas curtas, com foco em decisões do dia a dia ao manter aplicações PHP.

A ideia é reduzir fricção: configurações previsíveis, código legível, e um caminho claro para depurar quando algo quebra em produção.

Em DevOps, o segredo é repetição: builds determinísticos, ambientes equivalentes e deploys pequenos. Automação evita “mágicas” manuais.

Dica: quando possível, registre contexto (request id, user id, rota) — isso transforma logs em diagnóstico.

Checklist

  • Defina a hipótese (o que está ruim / o que você quer garantir).
  • Escolha a menor mudança que testa a hipótese.
  • Adicione uma verificação (teste, log ou métrica) para evitar regressão.
  • Documente a decisão em 3 linhas para o futuro.

Trecho de código

<?php
// Transação com rollback seguro
$pdo->beginTransaction();
try {
  $pdo->exec('UPDATE accounts SET balance = balance - 10 WHERE id = 1');
  $pdo->exec('UPDATE accounts SET balance = balance + 10 WHERE id = 2');
  $pdo->commit();
} catch (Throwable $e) {
  $pdo->rollBack();
  throw $e;
}
